# 佳乐考研 系统优化报告

## 1. 本轮目标

本轮没有把原有业务页面全部推倒重写，而是在保留已有知识库、闪卡、错题、模拟考试、资料管理等能力的基础上，优先重构最影响日常使用的“系统骨架”和学习闭环：

- 找得到功能；
- 知道下一步该做什么；
- 计划、专注和数据之间能够联动；
- 页面切换、刷新、移动端和后端异常时仍然可用；
- 后续扩展时不再依赖大量硬编码。

## 2. 已完成的核心改造

### 信息架构与操作逻辑

- 将原先平铺的约 23 个入口整理为“今日学习、知识学习、复习与训练、专项内容、更多”五组。
- 桌面侧栏支持折叠并持久化，移动端提供抽屉与底部导航。
- 页面位置写入 URL hash 与 localStorage，刷新后保持当前模块。
- 新增全局命令面板，支持 `Ctrl/Cmd + K`、`/`、Enter 与 Escape。
- 所有业务模块改为动态加载，并为每个页面增加独立错误边界。

### 学习首页

- 首页重构为学习驾驶舱，突出“下一步最佳行动”。
- 汇总今日任务、计划时长、专注进度、任务完成率、复习队列、连续学习、模考均分和学习健康度。
- 提供闪卡、知识复习、错题、知识库和个性化建议的快捷入口。
- 增加 skeleton、错误提示、重试和动态问候。

### 学习计划

- 后端 `/api/study-plans` 成为主数据源，localStorage 作为离线缓存。
- 首次进入先立即显示缓存，再与云端对账。
- 本地未同步任务自动补传，云端任务会规范化回写本地。
- 添加、完成、删除和 AI 批量生成均采用乐观更新，并在失败时回滚或保留本地副本。
- 计划变化通过全局事件即时刷新首页。
- 修复 AI 生成计划第 1 天错误落到“明天”的日期偏移问题。
- 增加同步状态提示、移动端布局、键盘关闭、遮罩关闭和可访问性标签。

### 全局专注计时

- 支持 25/50/90/120 分钟目标与科目选择。
- 支持开始、暂停、继续、完成并记录、重置。
- 计时状态可跨页面、刷新恢复。
- 后端可用时同步学习会话；后端不可用时仍允许本地专注。

### 个人设置

- 统一维护考试日期、每日学习目标、默认专注时长、目标院校/专业与四科目标分数。
- 设置变化会立即更新首页和侧栏倒计时。
- 支持本地数据导出、恢复默认偏好和安全清理缓存。
- 清理缓存时保留登录凭据，避免原实现 `localStorage.clear()` 导致意外退出。

### 后端与启动体验

- 修复学习会话开始/结束接口将 JSON body 误当 query 参数导致的 422。
- 健康检查增加数据库、AI Provider、模型和时间戳信息。
- FastAPI 启动事件迁移到 lifespan，消除弃用警告。
- 一键启动脚本支持端口冲突自动避让、动态 CORS 与动态前端 API 地址。
- 启动脚本只在环境不存在时安装依赖。

### 视觉、响应式与可访问性

- 建立统一的暗色玻璃拟态设计语言、卡片、按钮、状态色与层级。
- 完成 1024px 与 640px 关键断点适配。
- 增加 `focus-visible`、语义化 dialog、ARIA 标签和 `prefers-reduced-motion`。
- 新增应用图标并消除 favicon 404。

### 滚动与视口专项修复

- 将应用外壳严格约束在 `100vh/100dvh`，并为内容父级补齐 `min-height: 0` 与高度约束。
- 主内容区成为唯一有效的纵向滚动容器，支持鼠标滚轮、触控滑动和稳定滚动条。
- 顶栏保持固定，桌面侧栏与主内容互不抢占滚动。
- 移动端主内容增加底部安全留白，最后一段内容不会被固定底部导航遮挡。
- 修正知识库详情栏在小屏错误使用 `col-span-5` 导致的隐式多列问题；小屏为单栏，大屏为 3/4/5 栏布局。

### 综合知识库补全

- 新增幂等脚本 `backend/comprehensive_knowledge.py`，后端启动时自动补齐缺失章节、知识点、学习字段与轻量知识关系。
- 保留现有用户数据，不删除知识点，不重置掌握度、复习记录或学习进度。
- 数据规模由 39 章、295 点扩充为 44 章、642 点、335 条关系，空章节由多个降为 0。
- 学科分布：新传 24 章 / 435 点，政治 12 章 / 125 点，英语二 8 章 / 82 点。
- 642 个知识点全部具备摘要、核心内容、考试要点、答题模板和速记与辨析。
- 补齐研究方法、融合新闻实务、新传前沿、形势与政策、完形填空等原有缺口。
- 知识库 API 增加学科、章节和知识点数量字段，前端同步显示数量并优化三栏浏览体验。

### 642 个知识点全量深度重整

- 新增 `backend/deep_knowledge_upgrade.py`，对正式数据库中的 642 个知识点执行质量版本 v2 升级。
- 为知识点新增 `base_content`、`key_points`、`case_analysis`、`common_mistakes`、`training_steps`、`self_test` 和 `quality_version` 字段；原始讲义保存在 `base_content`，避免覆盖后无法追溯。
- 以 44 个章节画像和 38 类知识形态区分知识组织方式。传播模型强调要素、路径、反馈与环境；新闻实务强调任务、流程、成品与复核；历史类强调背景、事件、制度变化与影响；研究方法强调问题、测量、样本、效度与伦理；英语阅读、语法、写作、翻译和完形分别使用对应训练链。
- 每点独立生成深度讲义、关键要点、案例应用、六项易错辨析、六步训练、自测题、考试提示、六段答题结构和关键词记忆链，前端拆分为独立卡片展示。
- 后端启动顺序为“建表 → 综合补全 → 深度升级”，各阶段独立容错；质量版本达到 v2 后不会重复重写。
- 正式升级前使用 SQLite backup API 创建 `backend/exam_os.before_deep_upgrade_20260714_152930.db`。
- 正式升级后 642/642 点均为质量版本 v2，44 章、335 条知识关系不变，学习状态 SHA-256 校验前后一致。
- 全字段验收结果：正文最短 679 字、摘要最短 120 字、关键要点 296 字、案例 231 字、易错辨析 330 字、训练 363 字、自测 293 字、考试提示 295 字、答题模板 373 字、记忆提示 167 字；各字段均 642 条非空、642 条不同。
- 旧通用短语“先明确概念提出的背景……”“判断该原理在理论体系中的位置……”等出现次数均为 0。

## 3. 性能结果

## 5. 后续建议

以下工作适合作为下一阶段，而不是本轮收口的阻塞项：

1. 将剩余旧页面中固定的 `http://localhost:8000` 全部迁移到统一 API 客户端。
2. 为学习计划、闪卡、错题和模拟考试增加 Playwright 自动化回归测试。
3. 为 API 客户端减少 `any`，统一错误码与用户提示。
4. 调整 Docker Compose 为浏览器可访问的同源 `/api` 反向代理方案。
5. 初始化 Git 仓库并建立版本标签，便于回滚和继续迭代。
6. 在真实用户数据积累后，根据完成率、遗忘曲线和模考表现进一步优化推荐算法。

## 6. 说明

本轮重点完成了应用外壳、导航、首页、学习计划、专注计时、设置、关键后端接口、启动体验、视口滚动修复和综合知识库补全。原有闪卡、错题、模拟考试、资料管理等业务页面主体被保留，并接入新的统一外壳；不能表述为“所有业务模块已经全部重写”。

当前知识库已形成新传、政治、英语二的完整基础复习框架，但不能承诺覆盖未来所有考点。政治年度时政、院校自命题范围、参考书和招生要求具有动态性，应以报考年度教育部正式考试大纲、目标院校招生目录及官方通知为准。
