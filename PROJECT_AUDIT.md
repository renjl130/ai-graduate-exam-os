# Personal Study OS — 项目审计报告

> 审计日期：2026-06-09
> 目标：将聊天驱动的考研系统升级为知识库驱动的私人AI考研操作系统

---

## 一、当前架构

### 1.1 前端（Next.js 15）

- **框架**：Next.js 15 + React 19 + TypeScript + Tailwind CSS 4
- **路由**：App Router（单页面，Tab切换）
- **状态管理**：useState + localStorage
- **页面组件**：17个（Dashboard, AiTutor, Flashcards, StudyPlan, ResourceLibrary, AISearch, CoreTheories, HotTopics, MockExam, WrongQuestions, SchoolInfo, English, Politics, Vocabulary, Settings, Auth, KnowledgeBrowser, KnowledgeGraph）
- **API客户端**：完整封装（JWT认证、SSE流式、重试机制）

### 1.2 后端（FastAPI）

- **框架**：FastAPI 0.136 + Uvicorn
- **数据库**：SQLite（原生sqlite3，非ORM）
- **AI**：DeepSeek V3/R1（真实API调用，SSE流式）
- **认证**：自实现JWT（无外部依赖）
- **API端点**：50+ 个（含新增知识库20+个）

### 1.3 数据库

**已有表（10张）**：
- `users` — 用户
- `conversations` — AI对话会话
- `messages` — 对话消息
- `flashcards` — 闪卡（FSRS算法）
- `flashcard_reviews` — 闪卡复习记录
- `wrong_questions` — 错题
- `study_plans` — 学习计划
- `study_records` — 学习记录
- `exams` — 模拟考试
- `posts` — 社区帖子

**新增表（8张）**：
- `subjects` — 学科（新传/政治/英语）
- `chapters` — 章节
- `knowledge_points` — 知识点（核心）
- `knowledge_relations` — 知识点关系
- `exam_questions` — 真题
- `ai_summaries` — AI总结缓存
- `import_jobs` — 导入任务
- `study_sessions` — 学习会话

### 1.4 AI模块

- **AI服务**：`backend/services/ai_service.py`
- **支持模型**：DeepSeek V3（默认）、DeepSeek R1（复杂问题）、OpenRouter、Ollama
- **功能**：流式对话、资料整理、文件分析、试卷生成、学习计划生成
- **新增功能**：知识点AI讲解（详细/考法/精简三种风格）

### 1.5 知识库模块

- **数据源**：`data/` 目录下14个Markdown文件（约190KB）
- **覆盖学科**：传播学、新闻学、网络传播、广告学、公关学、新闻史
- **知识库脚本**：`backend/seed_knowledge.py`（自动提取→结构化存储）
